PIM Activator extension identity
Updated: 2026-06-03 (corrected to match the actually-deployed CRX)

Extension ID: eheocihmlppcophaeakmdenhgcookkab

This ID is deterministic from the public key embedded in manifest.json's
"key" field. Every install of this extension on any computer in any tenant
gets the same extension ID, which means:
  - The app registration redirect URI
    (https://eheocihmlppcophaeakmdenhgcookkab.chromiumapp.org/) is a single,
    well-known value
  - Customers can deploy the extension via Intune ExtensionInstallForcelist
    using the same ID everywhere
  - Deploy-PimActivatorBackend.ps1 doesn't need the operator to discover the
    ID manually via edge://extensions/

If you ever regenerate the signing key (don't, unless lost), the extension ID
will change. Update this file + every app reg's redirect URI + every
deployed forcelist entry.
